# 背景图位置调整设计

## 目标

在已有背景图自动 `cover` 裁切的基础上，允许用户分别调整图片的左右和上下焦点位置，并让预览与 PNG 导出保持一致。

## 交互设计

- 仅在已上传背景图时显示“图片位置”控件。
- 提供两条紧凑滑块：`左右` 与 `上下`，范围均为 0–100%。
- 左右 0% 表示靠左，50% 表示居中，100% 表示靠右；上下同理对应顶部、居中、底部。
- 默认值为 50% / 50%。上传或更换图片时恢复居中，移除图片时控件随之隐藏。
- 调整滑块后立即更新右侧预览；PNG 导出直接复用预览节点，因此自动保留相同位置。

## 数据与组件

- `BackgroundImageValue` 增加 `positionX`、`positionY` 两个数值字段，背景图片仍保持会话级状态，不写入持久化编辑器设置。
- 图片读取成功时由 `prepareBackgroundImage` 返回默认的 50 / 50。
- 新建 `BackgroundPositionControls`，只负责两条滑块的显示与变更回调。
- `EditorPanel` 在背景图存在时挂载位置控件，并通过现有 `onBackgroundImageChange` 更新完整背景图值。
- `PreviewCanvas` 将位置映射为图片的内联 `object-position: X% Y%`。

## 边界与错误处理

- 控件只产生 0–100 的整数；图片处理失败时保留当前图片及其当前位置。
- 更换图片成功后使用新图片和默认居中位置；旧对象 URL 的回收逻辑保持不变。
- 不增加缩放、旋转、拖拽、裁剪弹窗或背景图片持久化。

## 验证

- 单元测试覆盖控件回调、图片默认居中位置和会话状态更新。
- 应用测试覆盖上传图片后显示滑块、改变横纵值后 `object-position` 实时更新，以及移除图片后控件消失。
- 完整运行 Vitest、Pages 构建，并在桌面和窄屏浏览器检查布局、控制台和线上部署。
