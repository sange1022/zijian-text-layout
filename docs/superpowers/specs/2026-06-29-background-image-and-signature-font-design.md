# 背景图片与署名独立字体设计规格

## 目标

在不增加复杂操作的前提下，为排版工具增加自定义背景图片，并让署名位置和字体保持独立：背景图上传后自动适配当前画布；署名固定在用户选择的六个锚点之一，不因标题或正文长度变化而移动；标题、正文和署名均可使用汇文明朝体。

## 署名行为

- 署名继续使用左上、上中、右上、左下、下中、右下六个固定锚点。
- 署名使用绝对定位，与标题、正文排版容器分离。标题或正文增加、删除、换行时，署名坐标不变。
- 署名增加独立字体选择器，不再跟随正文字体变化。
- 署名文字、署名字体和六点位置选择在桌面左栏中保持同一紧凑行，不额外增加区块高度。
- 署名的颜色、字重和字号比例继续跟随正文样式，避免增加更多控制项；本次只将字体独立出来。
- 旧保存状态没有署名字体时，迁移为当时的正文字体，保证升级前后的视觉效果尽量一致。

## 字体

标题、正文和署名的字体菜单统一提供四项：

1. 思源宋体。
2. 思源黑体。
3. 得意黑。
4. 汇文明朝体。

当前电脑已安装本地字体 `Huiwen-mincho`（显示名称“汇文明朝体”）。应用通过 CSS `local('Huiwen-mincho')` 和 `local('汇文明朝体')` 加载，不复制或打包 24MB 的字体文件。找不到本地字体时回退到宋体类系统字体，不阻断编辑或 PNG 导出。

## 背景图片

- 在“画布背景”区加入简洁的“上传背景图”按钮；选择图片后显示文件名和“移除”按钮。
- 上传/移除入口与“画布背景”标题同一行，颜色控件保持原位，避免左侧面板因新增功能变高；文件名通过截断文本和悬停提示显示。
- 接受浏览器可解码的常见图片格式；文件超过 20MB 或无法解码时显示简短错误，不替换当前背景。
- 图片以会话内对象地址加载，不上传服务器，也不写入浏览器本地存储；刷新页面后背景图清除。
- 图片位于背景颜色之上、所有文字之下。背景颜色继续保留，作为图片透明区域或无图片时的底色。
- 图片采用 `object-fit: cover`、`object-position: center`：自动等比铺满画布，超出部分从四周居中裁剪，不拉伸、不留白。
- 切换任意尺寸预设时保留原图，并按新画布比例重新自动裁剪。
- 替换、移除图片或组件卸载时释放旧对象地址，避免持续占用内存。

## 预览与 PNG 导出

背景图片直接渲染在现有 `data-export-canvas` 节点内部，因此预览与 PNG 使用同一个 DOM 结构和同一套裁剪规则。图片成功解码后才进入画布；导出继续等待字体就绪，确保汇文明朝体和背景图片均按预览效果进入 PNG。

## 状态与组件边界

- `EditorState` 增加 `signatureFontId`，参与本地保存和旧状态迁移。
- 背景图片属于当前页面会话，由 `App` 管理对象地址和文件名，不进入持久化 `EditorState`。
- `BackgroundImageField` 负责文件选择、20MB 校验、解码、错误提示、替换和移除事件。
- `EditorPanel` 负责展示署名字体选择器和背景图片控件。
- `PreviewCanvas` 接收背景图片地址，在画布最底层渲染，并从 `signatureFontId` 解析署名字体。

## 错误处理

- 图片过大：显示“图片过大，请选择 20MB 以内的图片”。
- 图片无法读取或解码：显示“图片无法读取，请更换文件”。
- 字体不可用：静默使用系统宋体回退，编辑和导出保持可用。
- 选择无效图片时保留当前有效背景，不产生空白画布。

## 验收与测试

- 三个字体菜单均显示四个字体选项，汇文明朝体可用于标题、正文和署名。
- 改变正文字体不会改变署名字体；刷新页面后署名字体选择恢复。
- 旧状态自动补充署名字体，视觉上保持原有正文字体效果。
- 六个署名锚点在正文增删和换行后保持相同的边缘距离与水平对齐。
- 有效图片上传后位于文字下方，以居中 `cover` 方式铺满预览。
- 切换 3:4、1:1 和横版尺寸后，背景图均铺满且不拉伸。
- 移除背景图后恢复纯背景颜色；替换和移除会释放旧对象地址。
- 超过 20MB 或无法解码的文件显示错误并保留现有背景。
- PNG 导出包含背景图、署名位置和署名字体，效果与预览一致。
- 1440×900 桌面视口下左侧主要设置仍保持一页显示；390px 手机宽度下无横向溢出。

## 范围外

- 手动拖动背景图、缩放图片或选择焦点。
- 图片滤镜、透明度、旋转和多图层。
- 将上传图片持久化到刷新后的页面。
- 将字体文件上传到服务器或随应用分发。
