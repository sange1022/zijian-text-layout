# 背景图导出失败修复设计

## 问题与根因

上传背景图后，预览图片使用浏览器生成的 `blob:` URL。`exportPng` 调用 `html-to-image` 时启用了 `cacheBust: true`，该库会给待嵌入资源追加时间戳查询参数。普通 HTTP URL 可以接受查询参数，但 `blob:` URL 加上 `?时间戳` 后不再指向原对象，导致图片嵌入失败，导出长时间停留在“正在生成…”并最终显示失败。

没有背景图时不存在 Blob 图片资源，因此导出正常。这与用户观察完全一致。

## 修复

- 将 PNG 渲染选项的 `cacheBust` 改为 `false`。
- 保留背景图的 Blob URL，不在上传时额外转换 Data URL，避免重复占用大图片内存。
- 字体和站点静态资源使用带内容哈希或稳定路径，导出时不需要缓存破坏。
- 不改变图片上传、对象 URL 回收、PNG 尺寸和文件名逻辑。

## 验证

- 先增加失败测试，要求渲染器收到 `cacheBust: false`。
- 运行导出单元测试与完整测试套件。
- 在浏览器分别验证无背景图和有背景图两条导出路径，不再出现“生成失败”。
- Pages 构建通过后发布并在线复验。
