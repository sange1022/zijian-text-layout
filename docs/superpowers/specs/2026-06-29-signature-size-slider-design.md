# 署名字号滑块设计规格

## 目标

让署名字号可以独立调整，不再跟随正文字号变化，同时保持署名固定锚点、紧凑左栏和预览/PNG 一致。

## 控件与行为

- 在署名文字、字体和位置选择下方增加“署名字号”滑块。
- 滑块范围为 12–64px，步进为 1px，旁边实时显示当前数值。
- 新用户默认署名字号为 20px。
- 调整正文字号、正文内容或换行不会改变署名字号，也不会移动署名锚点。
- 署名颜色和字重继续跟随正文；字体、字号和位置保持独立。
- 署名字号变化立即更新预览，并使用同一数值进入 PNG 导出。

## 状态迁移

- `EditorState` 增加持久化字段 `signatureFontSize`。
- 旧保存状态没有该字段时，先把正文字号限制到 12–80px，再使用 `round(max(14, bodyFontSize × 0.72))` 迁移，最后限制到 12–64px。
- 当前状态中的非法值回退到默认状态；有效值在读取时限制到 12–64px。

## 布局

- 复用现有 `FontSizeSlider`，避免新增不同交互样式。
- 署名字号滑块使用单行紧凑布局，放在署名三列设置下方。
- 通过缩减文字内容区和署名区的少量垂直间距维持原有目标：1440×900 桌面视口下左栏主要设置完整显示，不出现纵向滚动。
- 390px 手机宽度下滑块占满可用宽度，不产生横向溢出。

## 验收与测试

- 滑块边界为 12 和 64，显示值与状态一致。
- 修改滑块后预览署名字号立即变化。
- 修改正文字号后署名字号保持不变。
- 旧状态迁移后保持接近升级前的署名大小。
- 刷新页面后恢复已选署名字号。
- 六个署名位置、独立字体、背景图和 PNG 导出行为不回归。
- 本地完整测试、Pages 构建、桌面/手机浏览器验收通过后推送 `main`，等待 GitHub Pages 自动部署，并复验公开链接。

## 范围外

- 独立署名字重、颜色、行高或字间距。
- 直接输入字号文本或增加字号预设按钮。
