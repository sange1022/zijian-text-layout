# 自定义画布大小设计

## 目标

在“常用尺寸”下方增加一行自定义画布宽高设置，同时保持现有五个预设、预览、背景图、文字排版与 PNG 导出行为不变。

## 交互

- 显示“自定义”标签、宽度输入、高度输入和“应用”按钮，布局为单行。
- 宽高单位为像素，允许 320–4096px 的整数。
- 用户点击“应用”或在输入框中按回车后切换到自定义画布。
- 超出范围的值在应用时限制到最近边界；空值或非数字沿用当前自定义值。
- 点击任一常用尺寸后切回该预设；自定义宽高继续保留，之后可再次应用。
- 当前为自定义画布时，预览下方显示“自定义 · 宽 × 高”。

## 状态与数据流

- `EditorState` 新增 `customWidth` 与 `customHeight`，默认均为 1080。
- `sizeId` 允许现有预设 ID 或 `custom`。
- 旧版持久化数据缺少自定义字段时自动补 1080 × 1080，不影响原来的预设选择。
- 新增纯函数 `getCanvasSize(state)`：自定义模式返回状态中的宽高，预设模式返回原 `SIZE_BY_ID` 数据。
- `App` 的 PNG 导出与 `PreviewCanvas` 都调用 `getCanvasSize`，确保预览尺寸、元信息和导出像素完全一致。

## 组件

- `SizePresets` 继续负责五个常用尺寸，并增加自定义宽高表单。
- `EditorPanel` 把自定义值和应用回调传给 `SizePresets`；应用时一次更新 `sizeId`、`customWidth`、`customHeight`。
- 不复制预设数据，不增加新的全局状态或依赖。

## 验证

- 状态测试覆盖旧数据迁移、自定义数据恢复和边界限制。
- 尺寸解析测试覆盖预设与自定义两条路径。
- 组件测试覆盖点击应用、回车应用和范围限制。
- 应用测试覆盖自定义预览尺寸、预览元信息以及切回常用尺寸。
- 完整运行前端测试、Pages 构建和桌面构建，并在桌面与窄屏浏览器检查布局。

## 范围外

- 不增加比例锁定、单位切换、预设保存、横竖互换按钮或超过 4096px 的超大画布。
